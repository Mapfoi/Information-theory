## LFSR File Encryptor (Lab 2)

Windows Forms application for **stream encryption/decryption** of any file using XOR with a key stream generated by a **Linear Feedback Shift Register (LFSR)**.

UI language is **Russian** (font size 14). All code comments are in English.

### LFSR parameters

- **Register length**: 28
- **Characteristic polynomial**: \(x^{28} + x^3 + 1\)
- **Seed**: entered manually in the GUI as a 28-character string of `0`/`1`

### How the algorithm works

The program implements a symmetric stream cipher:

1. Read the input file as a byte array.
2. Generate a key stream with the same length (in bits) as the file.
3. XOR the input bits with the key stream bits.
4. Save the result as a **binary byte stream**.

Because XOR is symmetric, the **same operation** is used for encryption and decryption:

- `cipher = plain XOR key`
- `plain = cipher XOR key` (with the same seed)

### Bit-level LFSR rule (as implemented)

- **Output bit**: MSB (most significant bit) of the register.
- **Shift direction**: shift left.
- **Feedback bit**: XOR of tapped bits for the polynomial terms **excluding** the highest term.

For \(x^{28} + x^3 + 1\) with the shift convention used in this project, feedback is:

- `feedback = state[25] XOR state[0]`
  - `state[0]` is MSB (output)
  - `state[27]` is LSB

Key stream is consumed as bytes, generated from 8 successive output bits.

### Short code excerpts

LFSR generation (feedback + shift + output):

```csharp
    public int NextBit()
    {
        var output = _state[0] ? 1 : 0;
        var feedback = _state[25] ^ _state[0]; //x^3 xor output

        // shift left: [0] <- [1] ... [26] <- [27], [27] <- feedback
        for (var i = 0; i < RegisterSize - 1; i++)
            _state[i] = _state[i + 1];

        _state[RegisterSize - 1] = feedback; //[27] = feedback

        return output;
    }
```

File processing (stream XOR):

```csharp
for (var i = 0; i < totalBytes; i++)
{
    var keyByte = lfsr.NextByte();
    outputBytes[i] = (byte)(inputBytes[i] ^ keyByte);
}
```

### Display of bits in the UI

The application shows:

- **Key stream bits**
- **Input file bits**
- **Output file bits**

In the text boxes, binary output follows this rule (the file on disk is always processed fully):

- If the file length is **at most 20 bytes**, the **entire** key / input / output is shown.
- If the length is **greater than 20 bytes**, only the **first 10** and **last 10** bytes are shown, with an omission line in between.

### Usage

1. Run the application.
2. Choose an input file (`Выбрать…`).
3. Enter the 28-bit seed (only `0` and `1` are allowed).
4. Choose where to save the output (`Сохранить как…`).
5. Click `Запустить шифрование/дешифрование (XOR)`.
6. To decrypt, run again and select the produced file as input with the **same seed**.
